Отличный вопрос! В Python разницы между двойными (") и одинарными (') кавычками нет — они работают абсолютно одинаково. Однако есть нюансы, которые стоит учитывать для удобства и читаемости кода.

1. Нет разницы в функциональности

Оба варианта создают строки одного типа (str):

name1 = "Алиса"    # Двойные кавычки
name2 = 'Боб'      # Одинарные кавычки
print(type(name1))  # <class 'str'>
print(type(name2))  # <class 'str'>

2. Когда что использовать?

a) Одинарные кавычки (')

  • Для коротких строк (по PEP 8 — стандарту стиля Python).

  • Если строка содержит двойные кавычки (чтобы не экранировать):

    text = 'Он сказал: "Привет!"'  # Удобно, не нужно экранировать "

b) Двойные кавычки (")

  • Если строка содержит апостроф (') (чтобы не экранировать):

    text = "It's a beautiful day"  # Удобно, не нужно экранировать '

  • Для f-строк (необязательно, но часто так делают):

    name = "Анна"
print(f"Привет, {name}!")  # f-строки обычно с "

c) Тройные кавычки (''' или """)

Используются для:
- Многострочных строк.
- Docstrings (описания функций/модулей).

Пример:

message = """Это
многострочный
текст."""

3. Экранирование кавычек

Если внутри строки есть такие же кавычки, их нужно экранировать (\):

escaped1 = 'Он сказал: \"Привет!\"'  # Экранирование "
escaped2 = "It\'s a cat"             # Экранирование '

Но лучше просто чередовать кавычки, чтобы избежать \:

clean1 = "Он сказал: 'Привет!'"    # Внешние ", внутренние '
clean2 = 'It\'s a cat'             # Или просто экранировать '

4. Стиль и PEP 8

Согласно PEP 8:
- Используйте одинарные кавычки (') по умолчанию.
- Двойные (") — если строка содержит ' или для f-строк.

Пример:

# PEP 8 рекомендует:
text1 = 'Простая строка'
text2 = "Строка с апострофом: it's"
text3 = f"f-строка: {value}"

5. Исключения

  • JSON требует двойных кавычек ("). Если работаете с JSON в Python, используйте ":

    import json
data = '{"name": "Алиса"}'  # JSON-строка
parsed = json.loads(data)   # Будет ошибка, если заменить " на '

Итог

  • Нет разницы между ' и " в Python.
  • Одинарные (') — для простых строк (PEP 8).
  • Двойные (") — если строка содержит ', или для f-строк.
  • Тройные ('''/""") — для многострочного текста.
  • Экранирование (\) — если внутри строки те же кавычки, но лучше чередовать.

Пример итогового выбора:

# Обычная строка
path = '/home/user/docs'

# Строка с апострофом
text = "Don't stop me now!"

# f-строка
name = "Иван"
greeting = f"Привет, {name}!"

# Многострочный текст
poem = """Ветер по морю гуляет
И кораблик подгоняет..."""
